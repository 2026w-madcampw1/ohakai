## Git Convention

### Branch Strategy

* `main`: 최종 제출용 브랜치
* `develop`: 기능 통합 및 테스트 브랜치
* `feat/기능명`: 세부 기능 개발 브랜치
* `fix/버그명`: 버그 픽스 브랜

### Commit Message

* `Feat`: 새로운 기능 추가
* `Design`: UI 레이아웃 및 디자인 변경
* `Fix`: 버그 수정
* `Docs`: 문서 수정 (README 등)
* `Chore`: 빌드 설정 및 라이브러리 관리
